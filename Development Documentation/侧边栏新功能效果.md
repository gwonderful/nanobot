# 侧边栏新功能效果

## 目标

侧边栏模仿 Codex 的项目化组织方式，让用户能清楚区分：

- 置顶的常用项目和对话。
- 有项目归属的工作空间对话。
- 没有项目归属的纯对话。
- 已归档的对话。

核心原则：

- 项目是对话的归属和组织方式，不等于对话本身，也不等于磁盘文件夹本身。
- 对话可以属于某个项目，也可以不属于任何项目。
- 置顶只改变侧边栏显示位置，不改变项目或对话的真实归属。
- 归档只作用于对话，不归档项目容器。
- 项目菜单里的“归档”是批量归档该项目下的未归档对话。
- 移除项目只移除侧边栏里的项目入口，不删除磁盘文件，不删除对话，不改变对话归属。
- 空项目可以显示，但添加空项目不自动创建对话。

## 核心模型

侧边栏分三段：

1. 置顶
2. 工作空间
3. 对话

### 工作空间

工作空间段下面是平级项目。

项目来源包括：

- 已有项目会话里出现过的项目。
- 用户从输入框选择已有项目或使用现有文件夹添加的项目。
- 用户从侧边栏“工作空间”标题右侧 `+` 显式添加的项目。
- 用户从“已归档对话”里取消归档项目对话后恢复出来的项目。

项目下展示该项目的未归档对话。

空项目也要能显示。空项目指用户显式添加过，但当前没有未归档对话的项目。

### 对话

没有选择项目的新对话是纯对话，显示在侧边栏“对话”段。

纯对话不显示在“工作空间”段。

“对话”标题右侧有 `+`，点击后进入空白新对话页，并且输入框不选择项目。

### 置顶

项目和对话都能置顶。

置顶规则：

- 置顶项目显示在“置顶”段，并带上该项目下的未归档对话。
- 置顶对话显示在“置顶”段，但它仍然属于原项目或纯对话。
- 置顶后的项目或对话不在原普通段重复显示。
- 如果一个项目已经置顶，该项目下的对话跟随项目出现在“置顶”段；项目内单个对话即使也被置顶，也不再额外重复显示一条。
- 置顶项目在“置顶”段里仍支持展开和折叠。

## 输入框交互

空白新对话页的输入框要有 Codex 风格的项目选择控件。

### 默认状态

默认显示“选择项目”。

此时直接发送：

- 创建纯对话。
- 新会话进入侧边栏“对话”段。

### 项目选择菜单

点击“选择项目”后打开菜单。

菜单包含：

- 搜索项目。
- 已有项目列表。
- 使用现有文件夹。
- 不使用项目。

项目搜索只搜索项目，不搜索对话。

搜索来源包括：

- 当前未归档项目。
- 只有归档对话的项目。
- 用户显式添加的空项目。

如果项目名重复，建议辅助显示短路径，避免用户选错。

### 选择已有项目

用户选择已有项目后：

- 输入框项目选择控件显示项目名。
- 首次发送后，新会话归到该项目下。
- 新会话显示在“工作空间”段对应项目里；如果该项目已置顶，则显示在“置顶”段的项目下。

### 使用现有文件夹添加项目

项目只能通过选择或输入现有文件夹来添加，不提供“输入项目名后自动创建默认文件夹”的路径。

添加现有文件夹有两种方式：

- 桌面软件环境中，不填写路径直接点击“使用文件夹”时，打开系统文件夹选择窗口。
- 普通浏览器环境中不能安全拿到本地目录绝对路径，因此保留手动输入现有文件夹绝对路径的方式。

选择或输入文件夹后：

- 文件夹成为一个项目。
- 项目出现在“工作空间”段。
- 输入框自动选中该项目。
- 首次发送后，新会话归到该项目下。

如果用户只添加项目但没有发送消息：

- 项目仍然显示为空项目。
- 不自动创建对话。

### 不使用项目

点击“不使用项目”后：

- 清空当前项目选择。
- 输入框回到“选择项目”状态。
- 直接发送会创建纯对话。

### 从侧边栏进入新对话

从侧边栏某个项目右侧 `+` 新建：

- 跳回空白新对话页。
- 输入框项目选择自动变成该项目。
- 用户发送后，新对话归到该项目下。
- 点击 `+` 本身只创建草稿状态，不真正创建会话；只有首条消息发送后才创建会话。
- 这个预选项目不能被跳转到空白页时的默认清空逻辑覆盖。

从侧边栏“对话”标题右侧 `+` 新建：

- 跳回空白新对话页。
- 输入框不选择项目。
- 用户发送后，新对话进入“对话”段。

从左上角“新对话”进入：

- 跳回空白新对话页。
- 输入框不选择项目。

### 已存在会话

已经存在的会话里，项目归属不应该随便切换。

进入已有会话后：

- 输入框可以显示当前项目和权限信息。
- 项目选择主要只用于空白新对话首发前。
- 不提供明显的“把已有会话改归属到另一个项目”的主路径。
- 如果后续确实需要改归属，应该作为单独功能讨论，不混在这次侧边栏改造里。

## 侧边栏交互

### 工作空间段

“工作空间”标题右侧有 `+`。

点击后打开添加项目菜单。

添加项目方式和输入框里的“使用现有文件夹”一致。

这里的“添加项目菜单”不能只等同于直接调起原生文件夹选择器：

- 只支持添加现有文件夹，不支持输入项目名自动创建默认文件夹。
- 桌面软件环境中，未填写路径时点击“使用文件夹”打开系统文件夹选择窗口。
- 在没有桌面宿主文件夹选择能力的浏览器环境里，也不能点击后无反馈；至少要提供手动输入现有文件夹路径的入口。
- 选择或输入文件夹后只添加项目入口，不创建对话。

添加后：

- 项目出现在“工作空间”段。
- 不自动创建对话。
- 不自动进入该项目的新对话，除非用户明确从项目右侧 `+` 新建。

### 项目行

项目行展示项目名。

项目可以折叠和展开：

- 展开时显示该项目下的未归档对话。
- 折叠时隐藏该项目下的对话。

项目行右侧保留两个入口：

- `+`：创建该项目下的新对话草稿。
- `...`：打开项目菜单。

点击项目行右侧 `+`：

- 跳回空白新对话页，并预选该项目。
- 只有用户发送首条消息后，才真正创建对话。

项目右侧按钮平时可以弱化或隐藏，hover / focus 时显示；触屏或无 hover 场景下必须仍能通过稳定入口打开。

### 项目菜单

项目菜单至少包含：

- 重命名。
- 置顶 / 取消置顶。
- 归档。
- 移出侧边栏。

项目菜单里的“归档”不是归档项目容器，而是归档该项目下所有未归档对话。

如果项目下没有可归档对话：

- “归档”显示为灰色。
- 不可点击。

如果项目下有可归档对话，点击“归档”需要弹窗确认：

- 标题说明将归档几个对话，例如“归档 1 个对话？”。
- 正文说明这些对话之后可以在设置的“已归档对话”中找到。
- 取消按钮使用“取消”。
- 主按钮使用“全部归档”。

确认后：

- 将该项目下所有未归档对话标记为已归档。
- 项目本身不进入归档状态。
- 该项目会变成空项目并继续留在侧边栏。
- 如果该项目是置顶项目，项目仍留在“置顶”段，但下面没有未归档对话。

项目重命名：

- 只改变 UI 展示名。
- 不改变项目路径。
- 不改变已有对话归属。

项目置顶：

- 整个项目块移动到“置顶”段。
- 项目下未归档对话跟随项目一起显示。
- 普通“工作空间”段不再重复显示同一项目块。

### 项目移除

项目移除建议文案使用“移出侧边栏”，不要写“删除项目”。

移除项目的含义：

- 从侧边栏移除该项目入口。
- 该项目下未归档对话不移动到“对话”段。
- 不把项目会话改成纯对话。
- 不删除本地项目文件。
- 不删除对话文件。
- 不归档对话。
- 不改变对话原有项目归属。

移除空项目：

- 项目入口直接从侧边栏消失。
- 效果上接近“删除这个空入口”，但文案仍应避免“删除项目”。

移除非空项目：

- 允许作为批量整理项目入口的动作，但必须二次确认。
- 确认弹窗要明确说明“这会从侧边栏移除该项目和它下面的对话入口，磁盘上的文件不会删除，对话也不会被删除或归档”。
- 移除后，该项目和其未归档对话不出现在主侧边栏。
- 推荐恢复方式：从“工作空间 +”重新添加同一个文件夹，项目和其未归档对话重新显示在项目下。

如果项目被移除后，设置里的“已归档对话”仍有该项目的归档对话：

- 已归档对话继续留在设置里。
- 已归档对话继续按原项目分组显示。
- 不会因为项目被移除而自动取消归档。
- 用户在设置里对其中某条对话点击“取消归档”时，该项目自动重新出现在主页侧边栏，该对话回到该项目下。
- 这个过程只是从设置转回主页，归属不变。

### 对话段

“对话”标题右侧有 `+`。

点击后：

- 跳回空白新对话页。
- 输入框不选择项目。
- 发送后创建纯对话。

### 对话行

首页侧边栏里的对话行需要隐藏快捷按钮，避免右侧过于复杂。

推荐规则：

- 对话行 hover / focus 时显示“置顶 / 取消置顶”快捷按钮。
- 对话行保留 `...` 菜单入口。
- 归档放在 `...` 菜单里，不额外放一个常驻快捷按钮。
- 右键菜单可做可不做；如果做，内容和 `...` 菜单一致。
- 触屏或无 hover 场景下，`...` 菜单入口必须可用。

对话菜单保留现有能力：

- 重命名。
- 置顶 / 取消置顶。
- 归档。
- 删除。

对话置顶后：

- 显示在“置顶”段。
- 仍保留原项目归属或纯对话归属。
- 不在原普通段重复显示。

对话归档后：

- 从侧边栏普通列表中消失。
- 如果原来置顶，也从“置顶”段消失。
- 不删除对话文件。
- 可在设置里的“已归档对话”中找回。

对话删除后：

- 删除会话。
- 从所有列表中移除。

## 已归档对话

归档功能模仿 Codex，入口放在设置里。

设置里新增一栏“已归档对话”。

不再把“显示归档 / 隐藏归档”作为侧边栏主入口。

“已归档对话”中展示所有已归档对话。

顶部工具区包含：

- 搜索已归档对话。
- 对话范围筛选，文案使用“全部对话”。
- 项目筛选，默认文案使用“所有项目”。
- 项目筛选里需要包含“无项目对话”或“纯对话”，用于筛选没有项目归属的归档对话。
- 右上角危险按钮使用“删除全部已归档对话”，不要只写“全部删除”。

建议按归属分组：

- 项目对话显示在对应项目名下。
- 纯对话显示在“对话”分组下。

项目分组行展示：

- 文件夹图标。
- 项目名。
- 该项目下已归档对话数量。
- `...` 菜单入口。

项目分组菜单只放危险操作：

- “删除此项目的已归档对话”。
- 或按数量显示为“删除 3 个已归档对话”。

不要使用“删除项目中的全部内容”这类文案，避免用户误以为会删除本地项目文件或项目里的未归档内容。

项目分组菜单不提供“取消归档全部”，避免一次性把一组归档对话全部恢复到主页侧边栏。

每条已归档对话只提供右侧两个操作：

- 取消归档。
- 删除。

交互规则：

- 两个按钮在桌面视觉上平时隐藏；它们存在于 DOM 或可访问性树里不算视觉可见。
- hover / focus 该行时显示。
- 触屏场景下需要能通过点击行、长按或稳定菜单显示操作。
- 已归档对话行本身不作为直接打开入口，避免打开时到底算归档状态还是恢复状态产生歧义。

取消归档后：

- 对话从设置的“已归档对话”中移出。
- 对话回到原来的项目或“对话”段。
- 如果原项目已经被移出侧边栏，取消归档时自动重新显示该项目，对话回到该项目下。
- 这个过程只是从设置转回主页，项目归属不变。
- 如果该对话原来是置顶对话，归档时已从置顶显示中移除；取消归档后建议第一期不自动恢复置顶。

删除后：

- 走现有会话删除逻辑。
- 彻底移除会话。

批量删除时：

- 点击“删除全部已归档对话”需要二次确认。
- 点击“删除此项目的已归档对话”也需要二次确认。
- 确认弹窗必须明确说明只删除已归档对话，不删除磁盘上的项目文件。

## 空项目规则

空项目可存在于“工作空间”段或“置顶”段。

空项目出现来源：

- 用户从侧边栏添加项目。
- 用户从输入框使用现有文件夹添加项目但还没发送消息。
- 用户显式添加过的项目，其下对话后来全部被归档或删除。
- 用户对某个项目执行批量归档后，该项目变成空项目。

空项目支持：

- 重命名。
- 置顶 / 取消置顶。
- 移出侧边栏。
- 从项目右侧 `+` 创建该项目下的新对话草稿。

空项目不支持：

- 归档。

## 文案区分

需要明确区分三个概念：

归档：

- 只对对话生效。
- 从主页侧边栏隐藏。
- 进入设置的“已归档对话”。
- 可以取消归档。

移出侧边栏：

- 只对项目入口生效。
- 不删除本地文件。
- 不删除对话。
- 不归档对话。
- 不改变对话归属。
- 不把项目会话移动到纯对话。

删除：

- 对对话是永久删除会话。
- 对已归档对话也是永久删除会话。
- 不用于删除项目文件。
- 项目入口如果需要“删除空项目入口”的效果，也推荐仍写“移出侧边栏”或“移除项目入口”。

## 多语言文案

新增文案必须保持 UTF-8 正确显示。

- 中文和英文文案优先准确。
- 其他语言如果暂时没有可靠翻译，可以回退为英文文案。
- 不允许出现 `?` 替代非 ASCII 字符的乱码文案。

## 状态优先级

对话显示位置按以下规则判断：

1. 已删除：不显示。
2. 已归档：只显示在设置的“已归档对话”里。
3. 所属项目已置顶：显示在“置顶”段的项目下。
4. 自身已置顶：显示在“置顶”段。
5. 有项目归属：显示在“工作空间”段对应项目下。
6. 无项目归属：显示在“对话”段。

项目显示位置按以下规则判断：

1. 已置顶：显示在“置顶”段，带上其下未归档对话。
2. 用户显式添加过：即使没有未归档对话也显示在“工作空间”段。
3. 有未归档项目对话：显示在“工作空间”段。
4. 只有归档对话，且未被显式添加：普通侧边栏不显示，只在“已归档对话”里作为分组出现。
5. 被移出的项目：普通侧边栏不显示；如果之后取消归档该项目下的对话，或重新添加同一路径项目，则项目重新出现。

## 非目标

当前不做这些能力：

- 不做独立的后端 Project 实体。
- 不允许随意把已有会话改到另一个项目。
- 不把项目本身作为可归档对象。
- 不在添加空项目时自动创建空对话。
- 不做输入项目名自动创建默认文件夹。
- 不把项目下未归档对话解除项目归属并移动到“对话”段。
- 不新增复杂标签系统。
- 不做项目级“取消归档全部”。

## 最终效果

用户可以在空白页决定新对话是否属于某个项目。

用户可以在侧边栏按项目管理项目对话，也可以保留无项目纯对话。

用户可以置顶重要项目或对话，让它们移动到“置顶”段，但不会破坏原归属。

用户可以归档单个对话，也可以批量归档项目下的对话。

归档后的对话统一到设置里的“已归档对话”中管理和找回。

用户可以移出不想在侧边栏看到的项目入口，但不会删除本地文件，也不会把项目对话错误移动到纯对话。
